# Как включить регистрацию студентов

## Что уже добавлено

В проекте есть готовые экраны:

- вход по email и паролю;
- регистрация студента;
- имя, код класса и выбор аватара;
- подтверждение email;
- выход из аккаунта;
- отдельные локальные профили для разных пользователей;
- скрытие админки от обычных студентов;
- демо-вход без аккаунта.

Пока `config.js` не заполнен, сайт продолжает работать через кнопку **«Продолжить в демо-режиме»**.

## Настройка преподавателем

### 1. Создайте проект Supabase

Откройте Supabase, создайте новый проект и дождитесь его запуска.

### 2. Создайте таблицы и правила доступа

Откройте:

`SQL Editor → New query`

Скопируйте содержимое файла:

`backend/schema.sql`

Нажмите **Run**.

### 3. Укажите адрес сайта

Откройте:

`Authentication → URL Configuration`

В поле **Site URL** укажите:

`https://bolachkova004-droid.github.io/linguapolisprogramme/`

Этот же адрес добавьте в **Redirect URLs**.

### 4. Скопируйте данные проекта

Откройте:

`Project Settings → API`

Скопируйте:

- Project URL;
- Publishable key или anon key.

Откройте файл `config.js` и заполните:

```js
export const SUPABASE_CONFIG = {
  url: "https://ВАШ-ПРОЕКТ.supabase.co",
  publishableKey: "ВАШ_PUBLISHABLE_ИЛИ_ANON_KEY",
  teacherEmails: [
    "email-преподавателя@example.com"
  ]
};
```

В браузерный код нельзя вставлять `service_role` key.

### 5. Сделайте свой аккаунт преподавательским

Сначала зарегистрируйтесь на сайте как обычный пользователь.

Затем в Supabase откройте SQL Editor и выполните:

```sql
update public.profiles
set role = 'teacher'
where email = 'email-преподавателя@example.com';
```

## Как регистрируется студент

1. Студент открывает ссылку на Linguapolis.
2. Нажимает вкладку **«Регистрация»**.
3. Вводит имя, email и пароль от 6 символов.
4. При необходимости вводит код класса, например `ENGLISH-7A`.
5. Выбирает один из 10 аватаров.
6. Нажимает **«Создать аккаунт»**.
7. Если включено подтверждение email, открывает письмо и нажимает ссылку.
8. Возвращается на сайт и входит через вкладку **«Войти»**.
9. Выбирает наставника и начинает урок.

После входа аватар можно поменять, нажав на картинку профиля.

## Важное ограничение текущей версии

Supabase уже отвечает за регистрацию и вход. Игровой прогресс и ответы пока сохраняются в браузере конкретного устройства. Таблицы для облачного хранения уже есть в `schema.sql`, но синхронизацию ответов и прогресса нужно подключить отдельным следующим этапом.
