# Linguapolis v3

Обновлённая версия тренажёра разговорного английского.

## Как установить на GitHub Pages

1. Откройте репозиторий `linguapolisprogramme` на GitHub.
2. Удалите старые файлы `index.html`, `style.css`, `script.js`, `data.json`.
3. Загрузите из этой папки новые файлы с такими же названиями.
4. Нажмите **Commit changes**.
5. Подождите 1–3 минуты и откройте опубликованный сайт.
6. Обновите страницу без кэша: `Ctrl + Shift + R` или `Command + Shift + R`.

Папку `backend` тоже можно загрузить, но она не влияет на работу GitHub Pages. В ней лежит схема будущей общей базы данных.

## Что исправлено

- защита от двойной отправки ответа и двойного начисления прогресса;
- проверка, что ответ действительно написан по-английски;
- слабые ответы больше не дают обязательный рост каждого навыка;
- черновик ответа сохраняется автоматически;
- приложение продолжает запускаться, если `data.json` временно не загрузился;
- более понятное сообщение при ошибке загрузки;
- безопасная работа при недоступном IndexedDB или localStorage;
- исправлено логирование закрытия завершённого урока;
- кнопки и навигация стали понятнее и доступнее с клавиатуры;
- прогресс навыков снова виден на телефоне.

## Что добавлено

- кликабельные подсказки-фразы;
- пример естественного ответа после проверки;
- персональная рекомендация в аналитике;
- тренд последних ответов;
- шкала этапов 25 / 50 / 75 / 100;
- база всех ответов в админке;
- фильтр ответов по профилю;
- экспорт ответов в CSV;
- импорт резервной копии JSON;
- индикатор онлайн/офлайн;
- ещё 3 сценария: кафе, нетворкинг и заселение в отель;
- production-схема PostgreSQL с попытками уроков и защитой от повторной награды.

## Хранение данных

Демо работает без сервера:

- профили и черновики — `localStorage`;
- ответы и журнал действий — `IndexedDB`;
- если хранилище браузера недоступно, приложение временно работает в памяти.

Данные разных устройств и браузеров не объединяются. Локальная админка не защищена настоящей авторизацией. Для работы с реальными учениками нужна серверная база и вход по аккаунту.

## Локальный запуск

Теперь `index.html` можно открыть двойным кликом: внутри есть резервная копия учебных данных. Для разработки удобнее всё же использовать локальный сервер:

```bash
python -m http.server 8000
```

Затем открыть `http://localhost:8000`.
